Vantagens e Desvantagens

thumb_up Vantagens

  • Portabilidade: As aplicações funcionam da mesma forma em qualquer ambiente que suporte Docker, eliminando o clássico problema de "na minha máquina funciona".
  • Eficiência e Leveza: Contêineres compartilham o kernel do sistema operacional do host, consumindo menos recursos (CPU, memória) que máquinas virtuais.
  • Implantação Rápida: Contêineres iniciam em segundos, pois não precisam carregar um sistema operacional completo, agilizando os ciclos de desenvolvimento e teste.
  • Isolamento: Aplicações e suas dependências são isoladas umas das outras, evitando conflitos de biblioteca e aumentando a segurança.
  • Escalabilidade: É simples criar e destruir contêineres rapidamente para atender a picos de demanda, facilitando a escalabilidade horizontal.

thumb_down Desvantagens

  • Curva de Aprendizado: Conceitos avançados como redes, volumes e orquestração exigem estudo e prática para serem dominados.
  • Segurança do Kernel: Como o kernel é compartilhado, uma vulnerabilidade no sistema operacional do host pode, teoricamente, afetar todos os contêineres.
  • Aplicações com Interface Gráfica: O Docker não é ideal para aplicações que dependem de uma interface gráfica do usuário (GUI), sendo focado em serviços de linha de comando.
  • Gerenciamento em Larga Escala: Gerenciar um grande número de contêineres manualmente é complexo, exigindo o uso de ferramentas de orquestração como Kubernetes ou Docker Swarm.
  • Desempenho: Embora mais rápido que uma VM, existe uma pequena sobrecarga de desempenho em comparação com a execução nativa de uma aplicação diretamente no servidor físico.
